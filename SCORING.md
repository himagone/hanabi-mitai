# スコアリングロジック

花火の観覧地点としての適性を 0〜100 点で評価する。

## 全体構造

```
totalScore = baseScore × distanceVisibility
```

- **baseScore**: 5つの評価項目の重み付き合計（0〜1）
- **distanceVisibility**: 距離に基づく乗数（0〜1）。遠すぎる地点のスコアを強制的に下げる

## 評価項目と重み

| 項目 | 重み | 説明 |
|------|------|------|
| 仰角 (viewingAngle) | 25% | 花火を見上げる角度が最適か |
| 標高 (elevation) | 15% | 打上地点より高い位置にあるか |
| 視線 (lineOfSight) | 35% | 建物に遮られていないか |
| 勾配 (slope) | 10% | 花火方向に開けた斜面か |
| 場所 (accessibility) | 15% | 公園・広場など観覧しやすい場所か |

## 各項目の詳細

### 距離視認性 (distanceVisibility)

花火の見かけの大きさと大気による減衰を考慮した乗数。全体スコアに掛け合わせる。

**見かけサイズスコア:**

花火の開花直径と距離から見かけの角度を計算。

```
見かけ角度(°) = atan2(開花直径, 距離) × 180 / π
```

- 3° 以上: 1.0（十分な大きさ）
- 0.3°〜3°: 線形補間
- 0.3° 以下: 0（ほぼ見えない）

**大気減衰スコア:**

```
atmosphericScore = exp(-max(0, 距離 - 3000) / 10000)
```

- 3km 以内: ペナルティなし
- それ以降: 指数的に減衰（晴天でも 15km 以上はかすみで劣化）

**最終値:**

```
distanceVisibility = sizeScore × atmosphericScore
```

**花火の開花直径:**

大会ごとに指定可能。指定がない場合はデフォルト 150m（5号玉相当）。

| 大会 | 最大号数 | 開花直径 |
|------|----------|----------|
| 足立の花火 | 4号 | 120m |
| 隅田川花火大会 | 5号 | 150m |

### 仰角スコア (viewingAngle)

花火の開花位置を見上げる角度が最適かを評価。

```
花火の高さ = 打上地点標高 + 開花高度
見上げ角度 = atan2(花火の高さ - 観覧者の目の高さ, 水平距離)
```

- 観覧者の目の高さ = 地面標高 + 1.5m
- 開花高度は直径から推定: `30 + 直径 × 1.1`（4号→162m、5号→195m）
- 最適角度: 35°（ガウス分布、σ=25°）
- 花火より高い位置にいる場合: 0.05（ほぼ最低点）

### 標高スコア (elevation)

打上地点との相対標高差をシグモイド関数で評価。

```
score = 1 / (1 + exp(-0.1 × 相対標高差))
```

- 打上地点より高い → 0.5 以上（高台ほど有利）
- 打上地点より低い → 0.5 以下
- ±30m 程度で飽和

### 視線スコア (lineOfSight)

観覧地点から花火の開花位置への視線が建物に遮られていないかを判定。

**方法（建物ポリゴン交差判定）:**

1. 観覧地点→打上地点への2Dレイ（視線）を生成
2. OSMから取得した建物ポリゴンのBBoxで高速フィルタ
3. レイと建物ポリゴンの各辺との2D線分交差判定
4. 交差点での建物頂上高さ（地面標高 + 建物高さ）と視線高さ（線形補間）を比較

**建物高さの推定（OSM データ）:**

| 建物タイプ | デフォルト高さ |
|------------|----------------|
| apartments / residential | 10m |
| house / detached | 7m |
| commercial / office | 15m |
| industrial / warehouse | 8m |
| garage | 3m |
| その他 | 8m |

`height` タグや `building:levels` タグがあればそちらを優先（1階=3m で換算）。

**スコア:**

- 遮蔽なし: 1.0
- 建物データなし: 0.8（不確実）
- 遮蔽あり: `0.2 × (1 - 遮蔽建物の割合)` — 1つでも遮蔽があると大幅減点

### 勾配スコア (slope)

地形の傾きが花火方向に向いているか（開けた斜面か）を評価。

**方法:**

1. 観覧地点の東西南北 ±33m の標高から勾配ベクトルを計算
2. 花火方向の単位ベクトルとの内積で方向の一致度を判定

**スコア:**

- 花火方向に下る斜面（開けている）: 0.5〜1.0
- 平坦: 0.5
- 花火方向に上る斜面（遮られやすい）: 0.0〜0.5

### 場所スコア (accessibility)

OSM の土地利用データから観覧のしやすさを判定。

| 場所 | スコア |
|------|--------|
| 公園・広場・河川敷・遊び場 | 1.0 |
| 住宅地 | 0.3 |
| データなし / その他 | 0.6 |

## 2パス方式（デスクトップのグリッド分析）

大量の候補地点を効率的に処理するため、2段階で評価する。

### パス1: 高速事前スコア (quickScore)

ネットワーク通信なしで計算可能な 3 項目のみで概算。

```
quickScore = (仰角×0.25 + 標高×0.15 + 場所×0.15) × distanceVisibility
```

全候補をこのスコアでソートし、上位 100 件を抽出。

### パス2: フルスコア (fullScore)

上位 100 件に対して視線チェック・勾配計算を追加で実行。

```
fullScore = (仰角×0.25 + 標高×0.15 + 視線×0.35 + 勾配×0.10 + 場所×0.15) × distanceVisibility
```

## データソース

| データ | ソース | 用途 |
|--------|--------|------|
| 標高 | 国土地理院 DEM5A（ズーム15、256×256 テキストタイル） | 標高スコア、仰角、視線、勾配 |
| 土地利用 | OpenStreetMap Overpass API | 場所スコア |
| 建物 | OpenStreetMap Overpass API | 視線チェック（ポリゴン交差判定） |

## おすすめ理由の生成

スコアの内訳と数値から短い説明を生成する。

| 条件 | 表示 |
|------|------|
| distance < 0.05 | 「遠すぎて視認困難」 |
| distance < 0.3 | 「距離が遠い」 |
| accessibility >= 0.9 | 「公園・広場」 |
| accessibility <= 0.2 | 「住宅地」 |
| lineOfSight >= 0.9 | 「遮蔽なし」 |
| lineOfSight >= 0.5 | 「見通し良」 |
| lineOfSight < 0.3 | 「建物で遮蔽あり」 |
| viewingAngle 25°〜45° | 「仰角良好」 |
| viewingAngle 10°〜25° | 「仰角やや低い」 |
| relativeElevation > 10m | 「+○m」 |
| relativeElevation > 3m | 「やや高台」 |
| slope > 0.7 | 「開けた方角」 |
